In the realm of AI and automation, handling documents and extracting information efficiently is of utmost importance.
The DocumentAgent introduces an agentic solution to this problem. It handles document ingestion and query tasks seamlessly by leveraging an internal swarm of agents to streamline document processing and information retrieval.

<Warning>
These tools are currently in our `experimental` namespace, indicating that we have tested the functionality but their interface may change. Please use them with that in mind and we appreciate any feedback on them.

If you do find any bugs please [log an issue](https://github.com/ag2ai/ag2/issues) in the AG2 repository. If you would like to add more tools or functionality, we would love your [contribution](https://docs.ag2.ai/docs/contributor-guide/contributing).
</Warning>


## Installation

Install AG2 with the `rag` extra to use the DocumentAgent.

```bash
pip install ag2[rag]
```

## Capabilities

The document agent could perform the following tasks:
1. Ingest documents from local or url. Supported formats:
    - PDF
    - IMAGE
    - DOCX
    - HTML
    - PPTX
    - ASCIIDOC
    - MD
2. Answer questions with RAG capability.

> **_NOTE:_**  currently, DocumentAgent answers questions only related to ingested documents.

## Internal Swarm Chat
The `DocumentAgent` leverages a swarm of internal agents to handle complex document processing tasks. Swarm chat allows multiple agents to collaborate and complete tasks efficiently. Here's a breakdown of how internal swarm chat is used within the `DocumentAgent`:

### Swarm Agents

The `DocumentAgent` orchestrates the following swarm agents:

- **Triage Agent**: Decides what type of task to perform from user requests.
- **Task Manager Agent**: Manages the tasks and initiates actions.
- **Data Ingestion Agent**: Ingests the documents.
- **Query Agent**: Answers user questions based on ingested documents.
- **Summary Agent**: Generates a summary of the completed tasks.

### Workflow

1. **Initialization**: The `DocumentAgent` initializes the swarm agents and sets up the context variables.
2. **Triage User Requests**: The `Triage Agent` categorizes the tasks into ingestions and queries.
3. **Task Management**: The `Task Manager Agent` manages the tasks and ensures they are executed in the correct sequence.
4. **Data Ingestion**: The `Data Ingestion Agent` processes the documents.
5. **Query Execution**: The `Query Agent` answers the user's questions.
6. **Summary Generation**: The `Summary Agent` generates a summary of the completed tasks.


## examples

```python
import autogen
from autogen.agents.experimental.document_agent.document_agent import DocumentAgent

config_list = autogen.config_list_from_json(
    "OAI_CONFIG_LIST",
    filter_dict={
        "model": ["gpt-4o"],
    },
)

llm_config = {
    "config_list": config_list,
}


document_agent = DocumentAgent(llm_config=llm_config)
document_agent.run(
    "could you ingest ../test/agentchat/contrib/graph_rag/Toast_financial_report.pdf? What is the fiscal year 2024 financial summary?"
)

```
