`#!python A2UIServer` wraps a plain `#!python autogen.beta.Agent` and produces a ready-to-serve [Starlette](https://www.starlette.io){.external-link target="_blank"} ASGI app that speaks **canonical A2UI** over HTTP. It mirrors `#!python autogen.beta.a2a.A2AServer`: hold the agent, configure A2UI with flat kwargs, call a `#!python build_*` method to get an app, and bring your own `#!python uvicorn` (or any ASGI server) to run it.

A2UI's wire is transport-agnostic, so two encodings of the same validated messages are offered:

- `#!python build_sse_app()` — Server-Sent Events (`#!python text/event-stream`), suited to browser `#!python EventSource` clients and reconnection.
- `#!python build_jsonl_app()` — canonical A2UI NDJSON (`#!python application/x-ndjson`), suited to generic A2UI clients that consume the native JSON Lines wire.

!!! note
    `#!python A2UIServer` lives in `#!python autogen.beta.a2ui.rest`, not the top-level `#!python autogen.beta.a2ui`. It depends on Starlette (declared as an additional dependency, not a core extra); a missing install surfaces as a clear hint rather than an opaque `#!python ImportError`.

## Minimal Server

```python linenums="1"
import uvicorn

from autogen.beta import Agent
from autogen.beta.a2ui.rest import A2UIServer
from autogen.beta.config import AnthropicConfig


def build_app():
    agent = Agent(
        name="ui_agent",
        config=AnthropicConfig(model="claude-sonnet-4-6"),
    )
    return A2UIServer(agent, protocol_version="v0.9").build_jsonl_app()  # POST /a2ui  →  A2UI NDJSON


app = build_app()

if __name__ == "__main__":
    uvicorn.run(app, host="127.0.0.1", port=8000)
```

The turn is served at `#!python POST /a2ui` by default; pass `#!python build_jsonl_app(path="/chat")` to change it. Swap `#!python build_jsonl_app()` for `#!python build_sse_app()` to stream Server-Sent Events instead.

## The Request Contract

The adapter speaks a minimal, dependency-free JSON body (no `#!python ag-ui` / `#!python a2a-sdk` types) so any client that can POST JSON can drive the agent:

```json
{
  "messages":  [{"role": "user", "content": "show a booking form"}],
  "variables": {"locale": "en"},
  "a2ui":      [{"version": "v0.9", "action": {"name": "confirm", "...": "..."}}],
  "a2uiClientCapabilities": {"v0.9": {"supportedCatalogIds": ["..."]}}
}
```

| Field | Meaning |
|---|---|
| `#!python messages` | The conversation. A trailing run of `#!python user` messages is the current turn; earlier messages become history. `#!python system` / `#!python developer` roles become prompt; `#!python assistant` becomes a prior response. |
| `#!python variables` | Context variables merged into the turn (e.g. locale, user id). |
| `#!python a2ui` | Client→server envelopes — button clicks (`#!python action`), v1.0 `#!python functionResponse`, and client errors — rewritten into corrective prompts for the turn. See [Actions](/docs/beta/a2ui/actions). |
| `#!python a2uiClientCapabilities` | Optional catalog negotiation, nested under the protocol version. Folded into the prompt so the LLM only targets components the client can render. |

A body that is not valid JSON, not a JSON object, or has a mistyped field returns **400** with an `#!python {"error": ...}` payload. A failure *during* the turn (after the 200 has been sent) surfaces as a final `#!python event: error` (SSE) or `#!python {"error": ...}` (NDJSON) frame and is logged — the connection is not torn down silently.

!!! note "The server is stateless"
    There is no session store: the client resends the full conversation on every request. Each turn runs on a fresh stream, so horizontal scaling needs no sticky sessions — but the client owns the history.

## Response Frames

Each turn streams the conversational prose first (when present), then one frame per validated A2UI message.

<Tabs>
  <Tab title="NDJSON">
```text
{"text": "Here is your booking form."}
{"version":"v0.9","createSurface":{"surfaceId":"s1","catalogId":"https://a2ui.org/..."}}
{"version":"v0.9","updateComponents":{"surfaceId":"s1","components":[...]}}
```
The optional leading `#!python {"text": ...}` line is AG2 framing (the prose), not itself an A2UI message. The remaining lines are canonical A2UI messages, one per line, ending at EOF.
  </Tab>

  <Tab title="SSE">
```text
event: text
data: {"text": "Here is your booking form."}

data: {"version":"v0.9","createSurface":{"surfaceId":"s1","catalogId":"https://a2ui.org/..."}}

event: done
data: {}
```
The prose arrives as an `#!python event: text` frame, each A2UI message as a default (unnamed) `#!python data:` frame, and the turn closes with `#!python event: done`.
  </Tab>
</Tabs>

When the agent answers in plain prose (or validation degrades after exhausting retries), only the prose frame is emitted — there are no A2UI message frames.

## Driving It From a Client

Any HTTP client works. Here is a full round-trip with `#!python httpx`, reading the NDJSON stream line by line:

```python linenums="1"
import json

import httpx


async def ask_ui(text: str) -> tuple[str, list[dict]]:
    prose, messages = "", []
    async with httpx.AsyncClient(base_url="http://127.0.0.1:8000") as client:
        async with client.stream("POST", "/a2ui", json={"messages": [{"role": "user", "content": text}]}) as resp:
            resp.raise_for_status()
            async for line in resp.aiter_lines():
                if not line:
                    continue
                frame = json.loads(line)
                if "text" in frame:
                    prose = frame["text"]
                elif "error" in frame:
                    raise RuntimeError(frame["error"])
                else:
                    messages.append(frame)  # a canonical A2UI message
    return prose, messages
```

Because the server is stateless, a follow-up turn just resends the prior messages plus the new one:

```python linenums="1"
json={
    "messages": [
        {"role": "user", "content": "show a booking form"},
        {"role": "assistant", "content": "Here is your booking form."},
        {"role": "user", "content": "make it for 4 people"},
    ]
}
```
