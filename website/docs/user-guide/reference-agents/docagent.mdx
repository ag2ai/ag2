# DocAgent: Comprehensive Documentation



## Introduction

**DocAgent** is a production-ready, agentic solution for document ingestion and information retrieval that leverages Retrieval-Augmented Generation (RAG) capabilities.
Built on AG2's multi-agent framework, DocAgent provides a streamlined, efficient architecture for processing documents and answering questions with natural language instructions.

[`DocAgent`](/docs/api-reference/autogen/agents/experimental/DocAgent)

### What Makes DocAgent Special?

- **Multi-Agent Architecture**: Orchestrates three specialized agents for optimal task processing
- **Concurrent Processing**: Handles document ingestion and query execution in parallel
- **Flexible Data Storage**: Supports both vector databases (Chroma) and in-memory storage
- **Comprehensive Format Support**: Processes 15+ document formats including PDFs, Office documents, images, and web pages
- **Citation Support**: Optional citation tracking with configurable chunk sizes
- **Production Ready**: Designed for real-world deployment scenarios with both vector and graph RAG capabilities

<Tip>
DocAgent is designed to be production-ready and supports your RAG needs with a simplified, efficient architecture. It uses either a local vector Chroma database (default) or an in-memory option (InMemoryQueryEngine) for flexible deployment scenarios. The graph RAG capabilities with Neo4j integration are production-ready and provide enhanced retrieval accuracy.

DocAgent continues to be developed and your feedback and [contributions](https://docs.ag2.ai/latest/docs/contributor-guide/contributing) are most welcome!
</Tip>

### Internal Agents

DocAgent orchestrates three specialized agents in a streamlined workflow:

#### 1. DocumentTriageAgent
- **Purpose**: Analyzes user requests and creates structured `DocumentTask` objects
- **Key Features**:
  - Uses Pydantic models for reliable task categorization
  - Structured output format with `ingestions` and `queries` lists
  - Natural language request interpretation
  - Handles both file paths and URLs

#### 2. TaskManagerAgent
- **Purpose**: Processes documents and queries using integrated tools
- **Key Features**:
  - Concurrent document ingestion with ThreadPoolExecutor
  - Concurrent query execution
  - Integrated tools: `ingest_documents` and `execute_query`
  - Support for multiple RAG engines (vector and graph)
  - Robust error handling and logging

#### 3. SummaryAgent
- **Purpose**: Provides comprehensive summaries of all completed operations
- **Key Features**:
  - Citation support with detailed formatting
  - Error reporting and status tracking
  - Structured output with numbered sections
  - Context-aware summarization

### Workflow

The DocAgent follows a linear, efficient workflow:

1. **Request Analysis**: The `DocumentTriageAgent` receives user requests and categorizes them into structured `DocumentTask` objects containing ingestions and queries
2. **Task Execution**: The `TaskManagerAgent` processes documents and queries concurrently using integrated tools:
   - `ingest_documents`: Handles document ingestion with concurrent processing
   - `execute_query`: Executes queries with optional citation support
3. **Summary Generation**: The `SummaryAgent` provides a comprehensive summary of all operations, including citations when enabled

## Data Stores & Query Engines

DocAgent supports multiple data stores and query engines for flexible deployment scenarios.

### Vector Store: Chroma Database

[Chroma](https://www.trychroma.com/) is a popular, open-source, vector database. DocAgent takes the Markdown and embeds and stores the data into the vector database.

**Key Features:**
- Persistent storage across sessions
- Efficient similarity search
- Configurable embedding functions
- Collection-based organization

**Requirements:**
- `OPENAI_API_KEY` environment variable (uses GPT-4o for embeddings)
- ChromaDB and LlamaIndex dependencies

**Usage:**
```python
# Default Chroma setup (automatic)
doc_agent = DocAgent(llm_config=llm_config)

# Custom collection
doc_agent = DocAgent(
    llm_config=llm_config,
    collection_name="my_documents"
)
```

### In-Memory Query Engine

DocAgent can also store the Docling Markdown in memory when digesting documents. When it comes time to query, DocAgent will inject the Markdown into the system message of an internal query agent and use an LLM to respond to the question.

**Key Features:**
- No external dependencies
- Full context utilization
- Cost-effective for small document sets
- Immediate availability

**Important Notes:**
- The full Markdown content from ingested documents is put into the context window
- Be cautious of putting too much content as it may exceed the LLM's context limits
- The Markdown is put into the start of the system message to maximize LLM caching
- As it's in-memory, ingested content will be lost when the agent is destroyed
- LLMs can be better at answering some queries than the vector store

**Usage:**
```python
from autogen.agents.experimental import DocAgent, InMemoryQueryEngine

# Create In-Memory Query Engine
inmemory_qe = InMemoryQueryEngine(llm_config=llm_config)

# Include the Query Engine when creating your DocAgent
doc_agent = DocAgent(
    name="doc_agent",
    query_engine=inmemory_qe,
    llm_config=llm_config,
)
```

## Document Processing Capabilities

### Document Parsing with Docling

Internally, DocAgent uses [Docling](https://github.com/DS4SD/docling) to convert documents into Markdown files, which are then ingested into the data store for querying.

**Supported Operations:**
- **File Processing**: Downloads and converts local files and URLs
- **Web Page Rendering**: Uses Selenium to browse and extract content from web pages
- **OCR Processing**: Extracts text from images using optical character recognition
- **Table Extraction**: Preserves table structure in Markdown format
- **Multi-format Support**: Handles all supported document formats

### URL Processing

DocAgent can process various types of URLs:

1. **Direct File URLs**: Downloads files directly (PDFs, DOCX, etc.)
2. **Web Pages**: Renders HTML content using Selenium
3. **Redirect URLs**: Follows redirects to get final content
4. **Binary Files**: Handles images and other binary formats

### Text Processing Pipeline

1. **Input Detection**: Determines if input is URL, file path, or directory
2. **Download/Read**: Retrieves content from source
3. **Format Detection**: Identifies document type and format
4. **Parsing**: Converts to Markdown using Docling
5. **Compression**: Applies LLMLingua text compression (for PDFs)
6. **Storage**: Saves processed content to specified location
7. **Indexing**: Ingests into configured RAG engines

## API Reference

### DocAgent Class

```python
class DocAgent(ConversableAgent):
    """The DocAgent is responsible for ingest and querying documents.

    Internally, it generates a group chat with a simplified set of agents:
    - TriageAgent: Analyzes user requests and creates DocumentTask
    - TaskManagerAgent: Processes documents and queries using integrated tools
    - SummaryAgent: Provides final summary of all operations
    """
```

#### Constructor

```python
def __init__(
    self,
    name: str | None = None,
    llm_config: LLMConfig | dict[str, Any] | None = None,
    system_message: str | None = None,
    parsed_docs_path: str | Path | None = None,
    collection_name: str | None = None,
    query_engine: RAGQueryEngine | None = None,
    enable_citations: bool = False,
    citation_chunk_size: int = 512,
    update_inner_agent_system_message: dict[str, Any] | None = None,
    rag_config: dict[str, dict[str, Any]] | None = None,
):
```

#### Key Methods

- `run(message: str, max_turns: int = 1)`: Execute document processing and querying
- `generate_inner_group_chat_reply()`: Internal method for orchestrating agent workflow

### DocumentTask Class

```python
class DocumentTask(BaseModel):
    """The structured output format for task decisions."""

    ingestions: list[Ingest] = Field(description="The list of documents to ingest.")
    queries: list[Query] = Field(description="The list of queries to perform.")
```

### Ingest and Query Models

```python
class Ingest(BaseModel):
    path_or_url: str = Field(description="The path or URL of the documents to ingest.")

class Query(BaseModel):
    query_type: QueryType = Field(description="The type of query to perform.")
    query: str = Field(description="The query to perform.")
```

## Troubleshooting

### Common Issues and Solutions

#### 1. Document Ingestion Failures

**Problem**: Documents fail to ingest
**Solutions**:
- Check file permissions and paths
- Verify document format is supported
- Ensure sufficient disk space
- Check network connectivity for URLs

```python
# Enable debug logging
import logging
logging.getLogger("autogen.agents.experimental.document_agent").setLevel(logging.DEBUG)
```

#### 2. OCR Issues with Images

**Problem**: Poor text extraction from images
**Solutions**:
- Ensure image quality is sufficient
- Try different image formats
- Check if image contains text
- Consider preprocessing images

#### 3. Memory Issues with Large Documents

**Problem**: Out of memory errors
**Solutions**:
- Use text compression (automatic for PDFs)
- Process documents in smaller batches
- Use Chroma database instead of in-memory
- Increase system memory

#### 4. Query Engine Connection Issues

**Problem**: Cannot connect to Chroma or Neo4j
**Solutions**:
- Verify database is running
- Check connection parameters
- Ensure proper authentication
- Use in-memory engine as fallback

#### 5. Citation Formatting Issues

**Problem**: Citations not displaying correctly
**Solutions**:
- Verify `enable_citations=True`
- Check `citation_chunk_size` setting
- Ensure document was processed with citation support
- Review citation formatting in system messages
